# Photo Quality Checker

Photo Quality Checker — це потужний Python-інструмент для комплексної автоматизованої оцінки якості великих масивів фотографій. Програма розроблена спеціально для e-commerce та контент-менеджерів, яким потрібно швидко перевіряти тисячі зображень на відповідність стандартам маркетплейсів.

## 1. Загальний опис програми

Photo Quality Checker — програма для автоматизованої перевірки якості великих масивів фотографій товарів. Вона орієнтована на контент-менеджерів та спеціалістів з e-commerce, яким необхідно швидко перевірити тисячі зображень на відповідність стандартам маркетплейсів (наприклад, Rozetka, Prom, Amazon).

### 1.1 Основні можливості

- Масова завантаження та аналіз зображень за URL або локальними шляхами.
- Перевірка роздільної здатності (ширина та висота у пікселях).
- Оцінка різкості фотографії методом Лапласіана.
- Виявлення некадрованих зображень (надмірні білі поля).
- Перевірка тіней і брудного фону на першому фото товару.
- Виявлення водяних знаків за шаблонами.
- Виявлення логотипів Rozetka на фото.
- Розпізнавання тексту: російська мова, URL, QR-коди (через OCR Tesseract).
- Виявлення тонкої рамки 1 пікс. по краях зображення.
- Перевірка прозорості (RGBA/PNG з альфа-каналом).
- Паралельна обробка до 16 потоків.
- Локальний кеш зображень для повторних запусків.
- Збереження результатів у файли Excel (.xlsx).
- Реагрегація вже готових результатів без повторної обробки.

### 1.2 Формат вхідних даних

Програма приймає файли Excel (.xlsx, .xls) або CSV (.csv), в яких є колонка з URL-адресами або локальними шляхами до фотографій товарів. Допускається кілька URL в одній комірці — розділених комою або переносом рядка. Також підтримуються шляхи до папок із зображеннями.

### 1.3 Формат вихідних даних

Після обробки програма генерує два Excel-файли у тій самій папці, де знаходиться вхідний файл:
- details_<назва_файлу>.xlsx — деталі кожного фото (метрики, статус, причина).
- status_<назва_файлу>.xlsx — зведений статус по кожному товару.


## 2. Опис інтерфейсу

Інтерфейс поділено на чотири «картки» (панелі) у лівій колонці та дві панелі — у правій.

### 2.1 Ліва колонка


#### ① Джерело даних

Панель для вибору вхідного файлу та колонки з URL-адресами.

| Елемент                   | Опис                                                                                                                              |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| Поле «Файл»               | Шлях до Excel (.xlsx/.xls) або CSV файлу. Можна ввести вручну або обрати через кнопку «Огляд…».                                   |
| Кнопка «Огляд…»           | Відкриває діалог вибору файлу. Після вибору програма автоматично читає заголовки колонок.                                         |
| Випадний список «Колонка» | Показує всі колонки файлу. Програма намагається автоматично визначити колонку з URL. Вибраний варіант зберігається між запусками. |



#### ② Вимоги до якості

Панель налаштування порогів метрик (детально — у розділі 4).

#### ③ Додаткові перевірки

Перемикачі необов'язкових аналізів та повзунки чутливості (детально — у розділі 5).

#### ④ Запуск

Кнопки керування обробкою та службові дії.

| Елемент             | Опис                                                                             |
| ------------------- | -------------------------------------------------------------------------------- |
| Потоків             | Кількість паралельних потоків (1 / 2 / 4 / 8 / 12 / 16). Рекомендовано 4–8.      |
| ▶ ЗАПУСТИТИ ОБРОБКУ | Розпочати / призупинити / відновити обробку. Кнопка змінює текст під час роботи. |
| ⬛ Стоп              | Зупинити обробку. Вже оброблені рядки зберігаються.                              |
| 📁 Папка             | Відкрити папку з результатами в провіднику файлів.                               |
| 🗑 Очистити кеш      | Видалити локальний кеш завантажених зображень (~/.photo_cache).                  |
| 🔄 Реагрегація       | Перерахувати зведений файл статусів на основі вже наявного файлу деталей.        |



### 2.2 Права колонка


#### Превʼю фото

Показує мініатюру першого завантаженого фото поточного товару під час обробки. Масштабується пропорційно. До початку обробки відображається заглушка.

#### Прогрес та лог


| Елемент                | Опис                                                                        |
| ---------------------- | --------------------------------------------------------------------------- |
| Прогрес-бар            | Горизонтальна смуга прогресу (0–100 %).                                     |
| Рядок статусу          | Текстовий статус: кількість оброблених/всього, відсоток, час, що залишився. |
| Поле пошуку 🔍          | Фільтр/пошук по журналу в реальному часі з підсвічуванням збігів.           |
| Кнопка ✕               | Очистити рядок пошуку.                                                      |
| Текстова область «Лог» | Журнал операцій: статус кожного фото, помилки, попередження.                |




## 3. Підготовка вхідних даних


### 3.1 Формат файлу

Програма підтримує такі формати:
- Excel: .xlsx, .xls
- CSV: .csv (будь-який роздільник, кодування UTF-8 рекомендовано)

### 3.2 Вимоги до колонки з фото

Колонка з посиланнями на фото може містити:
- Одне HTTP/HTTPS посилання на зображення.
- Кілька посилань, розділених комою (,) або переносом рядка (\n).
- Локальний шлях до файлу зображення.
- Локальний шлях до папки — програма підбере всі зображення у ній.
📝 Примітка: Програма автоматично визначає найбільш підходящу колонку. Ви також можете вибрати її вручну у випадному списку «Колонка».

### 3.3 Колонка ідентифікатора товару

Якщо у файлі є колонка «ID», «id», «goods_id», «код», «артикул» або «product_id» — вона використовується як ідентифікатор товару у звіті. Інакше використовується перша колонка.

### 3.4 Приклад структури файлу


| ID   | Назва товару      | goods_images_url                                                  |
| ---- | ----------------- | ----------------------------------------------------------------- |
| 1001 | Кросівки Nike Air | https://cdn.example.com/p1_1.jpg,https://cdn.example.com/p1_2.jpg |
| 1002 | Джинси Levi's 501 | https://cdn.example.com/p2_1.jpg                                  |
| 1003 | Куртка зимова     | C:\Photos\jacket\                                                 |




## 4. Налаштування вимог до якості

У панелі «② Вимоги до якості» налаштовуються порогові значення для оцінки кожного зображення. Кожен параметр має два значення: «хороше» (✔) та «погане» (✘). Зображення, які не потрапляють до жодної категорії, отримують статус «Середнє».

### 4.1 Параметри


| Параметр             | ✔ Хороше                                                        | ✘ Погане                                                         |
| -------------------- | --------------------------------------------------------------- | ---------------------------------------------------------------- |
| Ширина (px)          | Мінімальна ширина для «хорошого» зображення. За замовч.: 800 px | Максимальна ширина для «поганого» зображення. За замовч.: 500 px |
| Висота (px)          | Мінімальна висота для «хорошого» зображення. За замовч.: 800 px | Максимальна висота для «поганого» зображення. За замовч.: 500 px |
| Різкість (Laplacian) | Мінімальний індекс різкості для «хорошого». За замовч.: 50.0    | Максимальний індекс для «поганого». За замовч.: 20.0             |



### 4.2 Логічні оператори

Між показниками ширини та висоти діє логічний оператор, який задається окремо для «хорошого» і «поганого» сценарію.

| Оператор | Застосування до «хорошого»                            | Застосування до «поганого»                            |
| -------- | ----------------------------------------------------- | ----------------------------------------------------- |
| І (AND)  | Хороше, якщо ширина І висота ≥ порогу (обидві)        | Погане, якщо ширина І висота ≤ порогу (обидві)        |
| АБО (OR) | Хороше, якщо ширина АБО висота ≥ порогу (хоча б одна) | Погане, якщо ширина АБО висота ≤ порогу (хоча б одна) |


📝 Примітка: Рекомендовані налаштування для Rozetka: хороше ширина/висота = 800×800, логіка «АБО»; погане ширина/висота = 500×500, логіка «І»; різкість хороша = 50, погана = 20.

### 4.3 Статуси зображення


| Статус     | Умова                                                                |
| ---------- | -------------------------------------------------------------------- |
| ✅ Добре    | Усі метрики ≥ порогів «хорошого» (залежно від оператора)             |
| ⚠️ Середнє | Не відповідає «хорошому», але не є «поганим»                         |
| ❌ Погане   | Одна або більше метрик нижче порогу «поганого» (або виявлено дефект) |
| ⛔ Помилка  | Неможливо завантажити або обробити зображення                        |




## 5. Додаткові перевірки

Панель «③ Додаткові перевірки» містить необов'язкові аналізи. Вмикайте лише ті, що потрібні для вашого проєкту — кожна перевірка збільшує час обробки.

| Перевірка               | Що виявляє                                                                               | Параметр                               | За замовч.      |
| ----------------------- | ---------------------------------------------------------------------------------------- | -------------------------------------- | --------------- |
| Тіні / Брудний фон      | Перевіряє перше фото: фон має бути білим. Аналізує периметр (верх / низ / ліво / право). | Допуск 0–100 (0 = суворо, 100 = м'яко) | 50, вимкнено    |
| Некадровані (білі поля) | Виявляє надмірні білі поля навколо товару (зображення не займає весь кадр).              | Макс. % полів (1–50 %)                 | 15 %, увімкнено |
| Логотипи Rozetka        | Шаблонний пошук логотипів Rozetka на фото (OpenCV matchTemplate).                        | —                                      | Вимкнено        |
| Водяні знаки            | Виявляє водяні знаки за файлами-шаблонами з папки watermark_templates/.                  | —                                      | Вимкнено        |
| Російський текст        | OCR-аналіз: виявляє кирилицю, характерну для російської мови.                            | —                                      | Увімкнено       |
| URL / QR коди           | Виявляє текстові URL та QR-коди на зображенні (OCR + ZBar).                              | —                                      | Вимкнено        |
| Тонка рамка (1px)       | Виявляє чорну або темну рамку товщиною 1–2 px по самому краю фото.                       | —                                      | Вимкнено        |



### 5.1 Шаблони водяних знаків

Для перевірки водяних знаків додайте файли зображень-шаблонів (PNG/JPG/BMP) до папки watermark_templates/ у кореневому каталозі програми. Програма автоматично завантажить їх при старті.
⚠️  Увага: Перевірки з OCR (Tesseract) потребують налаштування Tesseract — дивіться файлу [docs/ocr.md](docs/ocr.md).

### 5.2 Допуск тіней

Повзунок «Допуск» (0–100) регулює чутливість виявлення тіней:
- 0–20 — суворо: будь-яке відхилення від ідеально білого фону вважається тінню.
- 30–70 — помірно: допускаються невеликі природні тіні.
- 80–100 — м'яко: тільки очевидно брудний фон.
📝 Примітка: Рекомендоване значення для Rozetka: 30–70.

### 5.3 Поріг некадрованих фото

Повзунок «Макс %» (1–50) задає максимально допустиму частку білих полів від ширини або висоти зображення. Наприклад, 15 % означає: якщо сума лівого і правого поля перевищує 15 % ширини, фото вважається некадрованим.


## 6. Запуск та керування обробкою


### 6.1 Покрокова інструкція

1. Відкрийте програму Photo Quality Checker.
1. У розділі «① Джерело даних» натисніть «Огляд…» та оберіть Excel або CSV-файл.
1. Переконайтеся, що у випадному списку «Колонка» вибрана правильна колонка з URL.
1. Налаштуйте пороги якості у розділі «② Вимоги до якості».
1. За потреби увімкніть необхідні перевірки в «③ Додаткові перевірки».
1. Виберіть кількість потоків (рекомендовано 4–8).
1. Натисніть «▶ ЗАПУСТИТИ ОБРОБКУ».
1. Спостерігайте за прогресом у правій колонці. Превʼю покаже перше фото кожного товару.
1. По завершенні програма повідомить про збереження файлів та відкриє діалог.

### 6.2 Пауза та відновлення

Під час обробки натисніть «⏸ ПАУЗА» для призупинення. Натисніть «▶ ПРОДОВЖИТИ» для відновлення з того місця, де зупинились. Всі вже оброблені результати зберігаються.

### 6.3 Зупинка

Натисніть «⬛ Стоп» для повного припинення обробки. Файли з частковими результатами будуть збережені у папці поряд із вхідним файлом.

### 6.4 Кількість потоків

Вибір оптимальної кількості потоків:

| Потоків | Сценарій                    | Примітка                                   |
| ------- | --------------------------- | ------------------------------------------ |
| 1–2     | Слабке підключення або тест | Мінімальне навантаження на мережу          |
| 4       | Стандартна робота           | Рекомендовано для більшості ПК             |
| 8       | Швидка мережа, 8+ ядер CPU  | Оптимальний баланс швидкості та ресурсів   |
| 12–16   | Серверна обробка            | Висока навантаженість, можливий throttling |



### 6.5 Кешування

Програма автоматично кешує всі завантажені зображення у папку .photo_cache/. Якщо той самий URL вже є в кеші, повторне завантаження не відбувається. Це значно прискорює повторний запуск.
Для примусового оновлення натисніть «🗑 Очистити кеш» перед запуском.

## 8. Конфігураційний файл

Всі налаштування автоматично зберігаються у файл config_photo_quality.json у кореневій папці програми. Редагувати файл вручну не обов'язково — інтерфейс синхронізується з ним при кожному запуску.

### 8.1 Структура файлу

```
{
  "good":   { "width": 800, "height": 800, "sharpness": 50.0 },
  "bad":    { "width": 500, "height": 500, "sharpness": 20.0 },
  "bad_logic_operator":  "І",
  "good_logic_operator": "АБО",
  "concurrency": 8,
  "last_manual_column": "goods_images_url",
  "border_ratio": 0.15,
  "shadow_threshold": 50,
  "options": {
    "check_logos":      false,
    "check_rus_text":   true,
    "check_shadows":    true,
    "check_qr_url":     false,
    "check_watermarks": false,
    "check_borders":    true,
    "check_ai":         false,
    "check_1px_border": false
  },
  "logo_match_threshold": 0.5
}
```


| Ключ                 | Опис                                             |
| -------------------- | ------------------------------------------------ |
| good / bad           | Пороги ширини, висоти та різкості                |
| *_logic_operator     | Логічний оператор «І» або «АБО»                  |
| concurrency          | Кількість паралельних потоків                    |
| last_manual_column   | Остання вибрана колонка з URL                    |
| border_ratio         | Поріг виявлення некадрованих фото (0.0–0.5)      |
| shadow_threshold     | Допуск виявлення тіней (0–100)                   |
| options.*            | Прапорці додаткових перевірок (true/false)       |
| logo_match_threshold | Поріг схожості для виявлення логотипів (0.0–1.0) |




## 9. Часті запитання та усунення неполадок


| Симптом                                       | Можлива причина                                    | Рішення                                                                        |
| --------------------------------------------- | -------------------------------------------------- | ------------------------------------------------------------------------------ |
| Кнопка «Запустити» не реагує                  | Не вибраний файл або колонка                       | Оберіть файл через «Огляд…» та переконайтеся, що вибрана колонка.              |
| Помилка «Не вдалося прочитати файл»           | Файл пошкоджений або відкритий в іншій програмі    | Закрийте Excel, перевірте формат файлу (.xlsx / .csv).                         |
| Зображення не завантажуються                  | Проблема з мережею або URL недійсний               | Перевірте підключення до інтернету та правильність URL у файлі.                |
| TesseractNotFoundError                        | Відсутній tesseract.exe у vendor/tesseract/        | Виконайте кроки з файлу [docs/ocr.md](docs/ocr.md) для встановлення Tesseract. |
| 'cannot find traineddata'                     | Відсутні .traineddata файли                        | Запустіть python scripts/fetch_tesseract.py.                                   |
| OCR повертає порожній результат               | Невірні мовні файли                                | Перевірте наявність eng/rus/ukr.traineddata у vendor/tesseract/tessdata/.      |
| Програма зависає або дуже повільно працює     | Увімкнені важкі OCR-перевірки або забагато потоків | Зменшіть кількість потоків, вимкніть «Російський текст» / QR-перевірки.        |
| Всі фото отримують статус «Погане»            | Надто суворі пороги якості                         | Знизьте пороги «хорошого» або підвищте пороги «поганого».                      |
| Файл результатів не з'являється               | Недостатньо прав або помилка збереження            | Переконайтеся, що папка з вхідним файлом доступна для запису.                  |
| Неправильна колонка вибрана автоматично       | Програма не розпізнала назву колонки               | Вручну виберіть правильну колонку в списку «Колонка».                          |
| Після «Реагрегації» файл статусів не оновився | Вибрано невірний файл деталей                      | Оберіть файл details_*.xlsx для конкретного продукту.                          |
| Помилка DLL на Windows                        | Відсутні DLL у vendor/tesseract/                   | Скопіюйте всі файли з Tesseract install directory, не тільки tesseract.exe.    |




## 10. Глосарій


| Термін                | Визначення                                                                                                                              |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Лапласіан (Laplacian) | Математичний оператор, що вимірює різкість зображення через дисперсію другої похідної яскравості. Чим вище значення — тим різкіше фото. |
| OCR                   | Optical Character Recognition — оптичне розпізнавання символів. Технологія перетворення зображень тексту у машиночитний формат.         |
| Tesseract             | Відкритий рушій OCR від Google. Підтримує десятки мов, у т.ч. українську, російську, англійську.                                        |
| Водяний знак          | Напівпрозоре або приховане зображення / текст на фото, що вказує на авторські права або джерело.                                        |
| Некадроване фото      | Зображення, де товар не займає весь кадр і є великі порожні (переважно білі) поля.                                                      |
| Кеш                   | Локально збережені копії завантажених зображень для пришвидшення повторних запусків.                                                    |
| URL                   | Uniform Resource Locator — адреса ресурсу в інтернеті (посилання на фото).                                                              |
| QR-код                | Двовимірний штрих-код, який закодовує текст або URL. Не допускається на фото товарів деяких маркетплейсів.                              |
| CSV                   | Comma-Separated Values — текстовий формат таблиці, де поля розділені комами.                                                            |
| PyInstaller           | Інструмент для пакування Python-програм у самостійні .exe-файли.                                                                        |
| Паралельні потоки     | Одночасна обробка декількох зображень для скорочення загального часу виконання.                                                         |
| Реагрегація           | Перерахунок зведеного файлу статусів на основі наявних деталей без повторного завантаження фото.                                        |
